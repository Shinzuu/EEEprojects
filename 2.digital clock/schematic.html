1. Overview & choices

MCU: ESP32 (NodeMCU-32S, 30-pin) — recommended over ESP8266

  • Hardware SPI runs the matrix at 10+ MHz cleanly (animations stay smooth).
  • Built-in WiFi for SNTP — no RTC chip needed if always near a network. Add DS3231 only if you want offline timekeeping.
  • Cheaper than the OLED-bundled 8266 board (450 BDT vs 2100 BDT). Pick the 30-pin Type-C variant — fits a breadboard with rails on both sides.
  • ESP8266 also works; pin assignments below cover both.

Display: MAX7219 4-in-1 (red), 32×8 dots

  • Pre-assembled module (SKU u3937, 460 BDT). 5-pin cable: VCC, GND, DIN, CS, CLK. Daisy-chain ready (DOUT pads on the right edge if you add more).
  • Library: MD_Parola + MD_MAX72XX (Arduino) or U8g2 (less idiomatic). Hardware = FC16 for these China-made boards (some are GENERIC; flip the constant if rows/cols are mirrored).
  • Pixel pitch ~3 mm; 4 modules = 128 mm wide. Fits HH:MM with colon as a 2-pixel block.

Power: 18650 → TP4056 charger → MT3608 boost → ESP32 + matrix

Single cell (3.0–4.2 V) charged over USB-C, boosted to 5 V for the matrix and the ESP32 5 V pin. ESP32 has its own AMS1117-3.3 onboard, so we don't need a separate 3.3 V regulator.

Brightness limit: the matrix at full brightness with all LEDs on draws ~600–800 mA. The MT3608 module is rated 2 A but the cheap inductors sag above ~1 A. Cap brightness at intensity = 4 in firmware (range 0–15) and you'll stay around 250 mA — comfortable for the cell and the boost converter.

Use the protected TP4056 variant (u101528, USB-C, 22 BDT) — has DW01 + 8205A under-voltage / over-discharge cutoff, which an ESP32 absolutely needs (it pulls 200+ mA bursts during WiFi transmit and will brown-out into a flat cell).

Runtime estimate: 18650 ≈ 2500 mAh × 3.7 V = 9.25 Wh. At 250 mA × 5 V = 1.25 W draw, runtime ≈ 7 h continuous. With brightness 1 + WiFi sleep between NTP syncs, push to 24 h+.

The matrix is 5 V logic but its DIN/CS/CLK are happy with 3.3 V swings (MAX7219's input threshold ≈ 3 V at VCC = 5 V) — no level shifter needed. Verified on hundreds of these builds.

4. Firmware sketch (Arduino, ESP32)

Libraries (Library Manager):

  • MD_Parola by majicDesigns
  • MD_MAX72XX by majicDesigns
#include <WiFi.h>
#include <time.h>
#include <MD_Parola.h>
#include <MD_MAX72xx.h>
#include <SPI.h>

#define HW_TYPE   MD_MAX72XX::FC16_HW    // FC16 for the China 4-in-1 boards
#define MAX_DEV   4
#define DATA_PIN  23
#define CS_PIN    5
#define CLK_PIN   18

const char* WIFI_SSID = "your-ssid";
const char* WIFI_PASS = "your-pass";
const char* TZ_INFO   = "+06";           // Asia/Dhaka, fixed offset; or use POSIX TZ string
const char* NTP1      = "pool.ntp.org";
const char* NTP2      = "time.google.com";

MD_Parola P = MD_Parola(HW_TYPE, DATA_PIN, CLK_PIN, CS_PIN, MAX_DEV);

void setup() {
  P.begin();
  P.setIntensity(2);                     // 0..15, keep low for battery + eyes
  P.displayClear();
  P.setTextAlignment(PA_CENTER);
  P.print("WiFi");

  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) delay(200);

  configTime(0, 0, NTP1, NTP2);
  setenv("TZ", "<+06>-6", 1); tzset();   // Bangladesh: UTC+6, no DST
  P.print("sync");
}

char buf[8];
void loop() {
  time_t now = time(nullptr);
  struct tm tm; localtime_r(&now, &tm);

  // alternate colon every 500 ms
  if ((millis() / 500) & 1) snprintf(buf, sizeof buf, "%02d:%02d", tm.tm_hour, tm.tm_min);
  else                      snprintf(buf, sizeof buf, "%02d %02d", tm.tm_hour, tm.tm_min);

  P.print(buf);
  delay(250);

  // dim at night based on hour (or read LDR on GPIO34)
  P.setIntensity(tm.tm_hour >= 22 || tm.tm_hour < 7 ? 0 : 3);
}

That's a working clock. Add a once-an-hour configTime() re-sync if you want lower drift; otherwise NTP at boot is enough — ESP32's RTC drifts ~1 s/day.

6. Enclosure / build notes

  1. Set MT3608 to 5.0 V before connecting the matrix. Out of the box it can be at 28 V. Disconnect the load, power from a 4 V bench supply or the cell, turn the brass pot CCW until the multimeter reads 5.0 V.
  2. Don't run the matrix at intensity 15 on battery — current spike is brutal on the boost. Use 0–4. Default to 2.
  3. Fit-check: the 4-in-1 matrix is ~128 × 32 mm. ESP32 + breadboard or a small protoboard tucks behind. Aim for a wood/3D-print front frame, dark acrylic or lightly tinted plastic in front of the LEDs for contrast.
  4. WiFi credentials: burn into firmware for v1. v2 — add a captive portal (WiFiManager library) so credentials can change without reflashing.
  5. Cable to matrix: the supplied 5-pin Dupont is fine for ≤30 cm. Longer runs: add 100 nF + 10 µF on matrix VCC at the matrix end to absorb switching transients.

7. Optional extensions

  • Date/temperature scroll — every 30 s the clock blanks, scrolls "Sat 26 Apr · 28°C", returns to time. P.displayText(...) handles scroll automatically.
  • Web UI — ESP32 hosts a small web page for setting brightness, alarm, color schedule. Use ESPAsyncWebServer.
  • Home Assistant integration — add ESPHome firmware instead of Arduino. The MAX7219 component has a built-in display driver.
  • Battery state-of-charge — ESP32 ADC on a divider (100 k / 100 k) from the cell; show "BAT" + percent every minute. Calibrate the ADC's nonlinearity with analogSetAttenuation(ADC_11db).
  • Deep sleep alarm clock mode — at midnight, ESP32 deep-sleeps; wake on RTC alarm or button. Drops standby to ~50 µA. Needs DS3231 alarm pin to GPIO33 (RTC_GPIO).
