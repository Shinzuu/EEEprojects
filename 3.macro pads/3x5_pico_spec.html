1. Why direct wiring (vs matrix)

AspectDirect (this build)4×4 matrix
GPIO used158 (4 row + 4 col)
Diodes (1N4148)016 (one per key, NKRO)
Ghostingimpossiblerequires diodes
NKROnativerequires diodes
Wiring complexity15 wires + 1 GND bus8 wires + 16 diodes soldered to switch legs
Firmware1 line per keymatrix scan loop
BOM costno diode cost+16× BDT 2 = ~32 BDT

You said you don't have the diodes (1N4148 / "Xenon" → likely heard "1N4148"). Direct wiring removes that requirement entirely. Pico has the pins — use them.

2. Pico pin budget

  • RP2040 exposes 26 usable GPIO: GP0–GP22 (23 pins) + GP26–GP28 (3 ADC-capable pins).
  • GP23, GP24, GP25, GP29 are tied to onboard functions (SMPS, VBUS sense, LED, VSYS divider) — leave alone.
  • 15 keys → 15 GPIO → 11 GPIO still free for future rotary encoder, OLED (I2C on GP4/GP5), RGB strip, etc.

                      Raspberry Pi Pico

   Q ────────────►│ GP0  (pin 1)            │
   W ────────────►│ GP1  (pin 2)            │
                  │      (pin 3)  GND ◄─┐   │
   E ────────────►│ GP2  (pin 4)        │   │
   R ────────────►│ GP3  (pin 5)        │   │
   T ────────────►│ GP4  (pin 6)        │   │
   A ────────────►│ GP5  (pin 7)        │   │
                  │      (pin 8)  GND ◄─┤   │
   S ────────────►│ GP6  (pin 9)        │   │
   D ────────────►│ GP7  (pin 10)       │   │   ← common GND bus
   F ────────────►│ GP8  (pin 11)       │   │     (1 wire from
   G ────────────►│ GP9  (pin 12)       ├───┘      every switch's
                  │      (pin 13) GND ◄─┤          2nd terminal)
   Z ────────────►│ GP10 (pin 14)       │
   X ────────────►│ GP11 (pin 15)       │
   C ────────────►│ GP12 (pin 16)       │
   V ────────────►│ GP13 (pin 17)       │
                  │      (pin 18) GND ◄─┘
   B ────────────►│ GP14 (pin 19)       │
                  │ ...                 │
                  │      (pin 40) VBUS  │ ← USB 5V (not used)
                  └─────────────────────┘

Each switch (B3F-4055, 4-pin tact):

           pin1 ──┐         ┌── pin3
                  │  switch │           pin1 ↔ pin2 internally
                  │  body   │           pin3 ↔ pin4 internally
           pin2 ──┘         └── pin4    closes pin1↔pin3 when pressed

   Wire ONE leg from one side  → GPIO (e.g. GP0)
   Wire ONE leg from other side → GND bus

   You only need 2 of the 4 legs. The other 2 are for PCB stability.
No pull-up resistors needed. Pico's RP2040 has internal pull-ups (~50 kΩ). Firmware enables them. Pin reads HIGH idle, LOW when pressed.

6. Firmware (KMK / CircuitPython)

Setup (one-time)

  1. Hold BOOTSEL on Pico, plug USB → mounts as RPI-RP2 drive.
  2. Flash CircuitPython UF2: circuitpython.org/board/raspberry_pi_pico. Drag UF2 onto the drive — Pico reboots as CIRCUITPY drive.
  3. Download KMK: github.com/KMKfw/kmk_firmware. Copy the kmk/ folder to CIRCUITPY/.
  4. Save code.py below to CIRCUITPY/code.py. Pico reboots and enumerates as a USB keyboard immediately.

code.py

import board
from kmk.kmk_keyboard import KMKKeyboard
from kmk.scanners.keypad import KeysScanner
from kmk.keys import KC

keyboard = KMKKeyboard()

# 15 keys, direct-wired, one GPIO per key, common GND.
# Order matches the visual layout: row1 left→right, row2, row3.
PINS = [
    board.GP0, board.GP1, board.GP2, board.GP3, board.GP4,   # Q W E R T
    board.GP5, board.GP6, board.GP7, board.GP8, board.GP9,   # A S D F G
    board.GP10, board.GP11, board.GP12, board.GP13, board.GP14,  # Z X C V B
]

keyboard.matrix = KeysScanner(
    pins=PINS,
    value_when_pressed=False,   # active-low (pin shorted to GND)
    pull=True,                  # enable internal pull-up
)

# Default keymap — replace with macros / media keys / mod combos as needed.
keyboard.keymap = [[
    KC.Q, KC.W, KC.E, KC.R, KC.T,
    KC.A, KC.S, KC.D, KC.F, KC.G,
    KC.Z, KC.X, KC.C, KC.V, KC.B,
]]

if __name__ == '__main__':
    keyboard.go()

Macro examples (drop in place of plain letters)

from kmk.modules.macros import Macros, Press, Release, Tap, Delay
from kmk.handlers.sequences import simple_key_sequence, send_string

macros = Macros()
keyboard.modules.append(macros)

# Ctrl+C, Ctrl+V on first two keys

# Type a string

# Media: play/pause, vol up, vol down

VUP  = KC.AUDIO_VOL_UP

    COPY, PASTE, KC.LCTL(KC.Z), KC.LCTL(KC.Y), SIG,
    KC.F13, KC.F14, KC.F15, KC.F16, KC.F17,
    PLAY,  VDN,    VUP,    KC.MUTE, KC.MEDIA_NEXT_TRACK,
]]

F13–F24 are real HID keys most OSes don't bind to anything by default — perfect for global hotkey targets in OBS, Davinci, browser extensions.

7. Build steps

  1. Mount switches in plate. Use the print+cut plate from 3x5Print+cut/plate_A4.pdf. B3F-4055 is friction-fit in 12.5 × 12.5 mm cutouts.
  2. Bend & bus the GND legs. Pick one corner pair on each switch (say bottom-left + bottom-right). Bend the chosen GND leg of every switch flat and solder a single bare wire across all 15 of them — that's your GND bus.
  3. Run the GPIO wires. Cut 15 lengths of hookup wire (~10 cm each). Solder one end to the OTHER leg of each switch, route to the Pico, solder to the assigned GP pin.
  4. Bus the Pico GND. One wire from any Pico GND pin (3, 8, 13, 18, 23, 28, or 38) → the switch GND bus.
  5. Flash firmware per §6.
  6. Test. Plug into PC. Open a text editor. Press each key in turn — should type Q W E R T / A S D F G / Z X C V B. Any dead key = check that switch's two solder joints.
Sanity check before powering up: with the Pico unplugged, multimeter in continuity mode — touch one probe to the GND bus, the other to each GPIO wire in turn. Should beep ONLY when you press the corresponding switch. If it beeps without pressing, you have a short.
